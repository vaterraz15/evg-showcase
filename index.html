The Platform

Thirteen modules. One coherent system.

Every module is its own surface, but they all share one schema, one set of design tokens, and route-level access control. Click any card to open that module in the demo below.

🏦

Unified Shell

/dashboard/overview

Navigation shell with a ⌘K command palette and operational metrics across every module.

📒

ClearLedger

/dashboard/ledger

Double-entry general ledger with journal entries, balanced postings, and a full audit trail.

🔁

Reconciliation

/dashboard/reconciliation

Multi-tier matching engine with exception queues that flag what arrived against what was expected.

⚖️

Resolution Engine

/dashboard/exceptions

Inline exception resolution with an ML-assisted feedback loop for continuous improvement.

🛡️

Dispute Center

/dashboard/disputes

Reg E, Visa VCR, Mastercard Chargeback, and Amex OMS dispute lifecycles with deadline tracking.

🤖

ML Classifier

/dashboard/ml

Transaction classification with opt-out controls and batch processing on the Anthropic API.

🔐

Sentinel

/dashboard/sentinel

SIEM, encryption-key management, RBAC audit, incident response, disaster recovery, SOC 2 posture.

🔎

Watchkeeper

/dashboard/watchkeeper

CIP, OFAC screening, SAR/CTR workflows, and a customer due-diligence review queue.

💳

Card Issuing

/dashboard/card-issuing

Marqeta Core v3: cardholders, cards, spend controls, and general-ledger linkage.

🚦

Rails Reference

/reference/rails

Rail-ID specification across NACHA, IMAD, ARN, TLID, and UETR identifiers.

🧩

Edge Cases

/reference/edge-cases

Network edge-case catalog covering Maestro NREC/NRID, MC EREC, Visa ARN, and Pulse RRN.

👤

Customer Portal

/portal

Small-business books: auto-categorization, deposit cadence, tax package, dispute intake.

👁️

Money Watch

/portal/money-watch

Expected-versus-arrived alerts on a business's own cash, with a 13-week cash projection.

About

Built solo.

EVF is a personal engineering project that shows full-stack work across the hardest parts of fintech: money movement, reconciliation, compliance, and card issuing. I built it as one system rather than a set of separate demos. This page is a public showcase. The source code and core algorithms stay private.

🧱

Systems architecture

Multi-tenant schema across 7 domains, event-driven jobs, route-level RBAC, and a shared design-token system for white-label theming.

💸

Payments & money movement

Eight payment networks modeled with correct rail identifiers, plus double-entry accounting that balances to the penny.

📋

Compliance & risk

Reg E disputes, OFAC/CIP screening, SAR/CTR workflows, SOC 2 posture, and an incident-response and disaster-recovery surface.
